The Capability Map

Six things SIO does — ~45 commands, one loop

Each cluster is a stage in the self-improvement loop. Hover a card to lift it; key commands are highlighted.

01
👁️
Observe

See what's going wrong

Mine every Claude Code session transcript for tool failures, repeated attempts, and user corrections — then cluster them into named error patterns and spot the workflows that work.

sio mine sio patterns sio errors sio flows sio differential-flows sio trend sio status
02
✍️
Suggest

Turn patterns into rules

Generate targeted CLAUDE.md rules from error clusters with a two-hop search (--refine / --strategy), review them interactively, then apply or roll back — every change reversible.

sio suggest sio suggest-review sio approve sio reject sio apply sio rollback sio promote-rule
03
🧠
Recall

Never re-solve a solved problem

Ask how a task was done before. SIO topic-filters distilled sessions, detects struggle→fix transitions, and surfaces the winning approach so the agent doesn't rediscover it from cold.

sio recall sio distill sio collect-recall sio briefing /done-before
04
🔬
Optimize · the DSPy brain

Make the rule-writer smarter

Compound-optimize the suggestion generator through a disciplined ladder — Bootstrap → AMPLIFY → MIPROv2 → GEPA — with cost gates, crash-safe resume, and content-addressable trainsets.

sio optimize-ladder sio optimize sio amplify sio curate sio train sio ground-truth sio promote-to-gold

✅

The optimizer ladder — discipline, not guesswork

SIO refuses to burn compute on the wrong rung. Each stage gates the next: cheap broadening before expensive search, data-size floors before MIPROv2, amplification before GEPA.

RUNG 1

Bootstrap

Few-shot from any trainset. Fast, runs on minimal labeled data.

RUNG 2

AMPLIFY

Paired-debate synthetic expansion to clear the row floor.

RUNG 3

MIPROv2

Few-shot + instruction optimization. Gated on valset size.

RUNG 4

GEPA

Reflective multi-step. Task/reflection LM split; ≥300-row floor.

Crash-safe · --resume-from skips completed rungs
Cost-gated · estimates & confirms before each run

Design stance

Measured assist, not autonomous override

Detection, generation, and outcome measurement are automatic. Application, deprecation, and final judgment stay human. SIO produces; you remain the editor.

AUTOMine sessions for errors
AUTOCluster & name patterns
AUTOGenerate candidate rules + measure outcomes
HUMANReview & approve each rule
HUMANApply, deprecate, final judgment
